# 更新日志

本文档记录 Mengnex 的重要变更。

## 0.1c - 2026-07-09

### 新增

- 新增 `start.ps1` 一键启动脚本，可在单个终端中同时启动前后端，并在脚本退出时一并停止两个进程。
- 设置页导航升级为一级分组加二级菜单结构，新增 `媒体库列表`、`扫描设置`、`元数据管理`、`系统日志`、`账号安全` 等入口。
- 新增全局扫描设置，可配置缩略图与预览图的最大分辨率和输出质量，并通过 `/api/preferences` 持久化到后端。
- 为照片查看器新增“预览图打底，原图从上到下渐进替换”的大图查看加载效果。

### 变更

- 移除照片显示来源的用户选择项，前端统一改为列表优先使用缩略图，其次预览图；大图查看优先使用原图。
- 设置页中的原“媒体库”调整为二级菜单 `媒体库列表`，原“偏好设置”调整为 `首选项` 分组下的二级菜单。
- 照片查看器由直接显示单一路径改为区分列表图、查看器预览图与原图来源。
- 缩略图与预览图的渲染参数不再写死在代码常量中，改为从应用级设置读取。

### 优化

- 前端主题系统升级，支持亮色、暗色、跟随系统，并为全局背景、导航、头部、侧边栏和任务页等界面补齐主题适配。
- 全局背景改为纯渐变方案，亮色使用抹茶粉渐变，暗色使用紫色渐变，并统一为左上到右下的方向。
- 媒体库管理页全面改造为 HeroUI 风格表单，修复输入框、下拉单选、主题色和长列表滚动表现。
- 媒体库卡片重做布局与主题适配，支持更紧凑的操作区、主题化标签、改进后的照片拼图封面以及可滚动的更多菜单。
- `/tasks` 页面文案已统一调整为中文。

### 修复

- 修复 PowerShell 启动脚本的编码与语法问题，避免 `try/catch` 解析失败和字符串乱码。
- 修复 HeroUI 选择器在错误用法下触发的 `cannot be rendered outside a collection` 等前端报错。
- 修复新建媒体库时输入框无法输入的问题，并统一改用稳定可控的 HeroUI 表单组合。
- 修复媒体库页下拉过长时超出屏幕但无法滚动选择的问题。
- 修复媒体库卡片在亮色主题下标签文字和背景不匹配的问题。
- 修复照片大图查看错误优先使用预览图的问题，恢复为预览图打底后切换原图的设计。

## 0.1b - 2026-07-09

### 新增

- 新增持久化任务表 `app_tasks`，统一承载扫描任务与缩略图/预览图生成任务。
- 新增任务控制接口，支持对后台任务执行暂停、继续、取消。
- 新增扫描任务的实时进度字段，包括 `processed_files` 与 `removed_files`。
- 新增 SQLite 运行时优化配置，包括 `WAL`、`busy_timeout` 与常用索引初始化。
- 新增照片列表与媒体列表的分页能力，后端支持 `limit` / `offset` 查询参数。
- 新增照片页分批加载能力，避免一次性拉取全量照片。

### 变更

- 将媒体库扫描改为真正的后台任务，请求会立即返回任务信息，而不是阻塞到扫描完成。
- 将缩略图生成任务从进程内内存状态迁移为数据库持久化状态，应用重启后任务历史仍可查看。
- 统一任务中心的数据来源，`/api/tasks` 现在直接读取持久化任务表，不再拼接内存任务。
- 媒体库删除前会检查是否仍存在运行中的后台任务，避免在任务执行中删除资源。
- 照片页与媒体库设置页调整为优先按批次加载照片数据，降低大库场景下的首屏压力。

### 优化

- 扫描时改为先按媒体库预加载已有文件索引，再进行路径比对，减少逐文件查库开销。
- 扫描结束后会自动清理磁盘上已删除文件对应的数据库记录与衍生缓存文件，避免索引长期脏化。
- 扫描写入 `media_items`、`media_files`、`photo_assets` 的多步数据库操作已纳入事务，降低半成功状态风险。
- 删除媒体库时，多表删除操作已纳入事务，降低中途失败后留下残缺数据的风险。
- 任务执行循环会感知任务状态，在暂停时等待，在取消时尽快安全退出。

### 修复

- 修复任务中心中扫描任务与缩略图任务状态来源不一致的问题。
- 修复首次创建媒体库后，前端仍按同步扫描完成来提示用户的问题。
- 修复扫描后不会清理失效文件记录的问题。
- 修复缩略图任务在应用重启后状态丢失的问题。

## 0.1a - 2026-07-08

### 新增

- 创建 Rust `api` 服务，包含 Axum、SeaORM、SQLite、`api/data` 数据目录、OpenAPI 输出、Swagger UI，以及面向 Web 客户端的 CORS 配置。
- 增加媒体管理的基础数据结构：媒体库、扫描任务、共享媒体项/文件，以及专用照片资源表。
- 增加照片媒体库的文件系统扫描能力，支持写入数据库并通过 API 端点流式读取本地媒体文件。
- 在 Web 应用中增加首选项和媒体库管理设置页。
- 增加 `@hey-api/openapi-ts` 生成流程与类型安全的前端 API 客户端。
- 更新照片图库页面，通过 API 加载已扫描照片，并使用现有图库与查看器组件展示。
- 新增应用级偏好设置存储与 API，可选择照片默认优先使用缩略图还是原图。
- 新增照片媒体库的 `thumb` / `preview` 缓存生成能力，缓存文件存放在 `api/data/thumb` 和 `api/data/preview`。
- 在 `photo_assets` 中新增缓存元数据字段，用于记录缓存路径、大小和生成时间。
- 新增媒体库手动生成缓存、删除缓存的能力。
- 新增统一任务中心页面，以及 `/api/tasks` 聚合接口，用于集中查看扫描任务和缓存生成任务。
- 新增媒体库设置、信息、删除等弹窗交互。
- 新增 WebP 缓存输出，`thumb` 与 `preview` 统一切换到 WebP 主格式。
- 新增缓存目录占位文件与 Git 忽略规则，避免把实际生成缓存提交到仓库。

### 变更

- API 的 UUID 改为以文本字符串写入 SQLite，避免数据库查看时出现二进制 UUID 内容。
- API 源码结构调整为 `core`、`infra` 和功能 `modules`。
- 将仓库忽略规则集中到根目录 `.gitignore`。
- 照片浏览逻辑改为支持缓存优先显示：列表可优先使用 `thumb`，查看器可优先使用 `preview`。
- 手动生成缓存改为后台任务执行，并支持进度跟踪，而不是阻塞在设置页面中。
- 媒体库卡片改为聚焦封面与操作，详细统计信息统一移动到“更多 -> 信息”弹窗中。
- 媒体库管理能力扩展为支持重新扫描、启用或停用、重命名、修改根路径，以及配置扫描后自动补齐缓存。
- 手动生成缓存默认只补齐缺失或过期资源，不再强制整库重建。
- 缓存编码从 JPEG 迁移为 WebP，以获得更好的体积和传输表现。
- 全局布局改为顶部头部固定、主内容区独立滚动，避免长内容把侧边栏一并拉伸。

### 修复

- 通过允许本地 IP 图片优化，修复 Next Image 无法加载本地 API 图片的问题。
- 修复前端对 hey-api 生成客户端响应的解包逻辑。
- 将设置页移动到 `(setting)` 路由组下，修复设置路由布局残留问题。
- 修复设置页文案回退为英文的问题，恢复中文界面文案。
- 修复媒体库卡片上操作图标的对齐和悬停态表现。
- 修复媒体库“更多”菜单在点击后打开弹窗时不自动收起的问题。
- 修复侧边栏底部导航位置，使 `任务` 和 `设置` 始终固定在最底部。
- 修复图库、任务列表等长内容页面会连带拉伸侧边栏和头部布局的问题。
